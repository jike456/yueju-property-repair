# 悦居物业报修系统 - 前端页面设计方案（Vue + Element Plus）

## 一、整体架构

### 1.1 技术栈
- Vue 3 + Element Plus
- Vue Router（路由）
- Pinia 或 Vuex（状态，存 token、用户信息）
- Axios（请求封装，统一加 Authorization）

### 1.2 角色与路由
- **业主**：登录、注册、首页、我的报修、报修详情、评价、消息
- **维修工**：登录、待办工单、工单详情、消息
- **管理员**：登录、用户管理、公告管理、工单管理、数据统计、消息

### 1.3 布局结构
- 顶部导航栏：Logo、菜单、用户头像下拉（退出）
- 左侧菜单：按角色显示不同菜单
- 主内容区：`router-view` 展示当前页面

---

## 二、公共页面

### 2.1 登录页
- **路径**：`/login`
- **布局**：居中卡片，背景可加渐变或图片
- **内容**：
  - 标题：「悦居物业报修系统」
  - 表单：账号（输入框）、密码（密码框）
  - 登录按钮
  - 底部链接：「业主/维修工注册」
- **交互**：登录成功后存 token、用户信息，按角色跳转首页

### 2.2 注册页
- **路径**：`/register`
- **布局**：居中多步骤表单
- **内容**：
  - 步骤 1：账号信息（用户名、密码、确认密码、真实姓名、手机号、身份证）
  - 步骤 2：角色选择（业主 / 维修工）
  - 步骤 3：业主填写楼栋号、单元号、房间号、面积、入住日期；维修工填写技能类型、工作年限
- **交互**：提交后跳转登录页

---

## 三、业主端页面

### 3.1 业主首页
- **路径**：`/owner/home`
- **布局**：上中下分区
- **内容**：
  - 顶部：欢迎语、快捷入口（我要报修）
  - 中部：最新公告（卡片列表，标题、摘要、发布时间）
  - 底部：我的工单入口、消息入口
- **交互**：点击公告可展开详情；点击「我要报修」进入报修页

### 3.2 我要报修
- **路径**：`/owner/repair/create`
- **布局**：表单页面
- **内容**：
  - 故障类型：级联选择器（一级 + 二级）
  - 报修标题：输入框
  - 故障描述：多行文本
  - 报修图片：上传组件（支持多图）
  - 预约时间：日期时间选择器
  - 详细地址：输入框（可选，默认业主地址）
  - 优先级：单选（低/中/高/紧急）
  - 提交按钮
- **交互**：提交成功后提示并跳转到「我的报修」

### 3.3 我的报修
- **路径**：`/owner/repair/list`
- **布局**：表格 + 筛选
- **内容**：
  - 筛选：状态下拉（全部/待派单/待处理/处理中/待确认/已完成/已取消）
  - 表格：工单编号、标题、故障类型、状态、优先级、创建时间、预约时间、维修工姓名、操作
  - 分页
  - 操作：查看详情、取消（仅待派单等可取消状态）
- **交互**：点击行或「查看」进入详情页

### 3.4 报修详情（业主视角）
- **路径**：`/owner/repair/detail/:id`
- **布局**：信息卡片 + 流程时间线
- **内容**：
  - 基础信息：工单编号、标题、故障类型、描述、图片、地址、状态、优先级、创建时间、预约时间
  - 处理过程：维修工姓名、派单/接单/开始/完成时间、备注、处理图片、使用材料
  - 评价信息：评分、内容、图片、评价时间（已评价时显示）
  - 操作：未评价且状态为待确认/已完成时显示「去评价」按钮
- **交互**：点击「去评价」进入评价页

### 3.5 提交评价
- **路径**：`/owner/repair/:orderId/evaluate`
- **布局**：表单页面
- **内容**：
  - 评分：星级评分（1–5）
  - 评价内容：多行文本
  - 评价图片：上传组件
  - 匿名：开关
  - 提交按钮
- **交互**：提交成功后返回工单详情

---

## 四、维修工端页面

### 4.1 维修工首页 / 待办工单
- **路径**：`/repairman/orders`
- **布局**：表格 + 筛选
- **内容**：
  - 筛选：状态下拉（待处理/处理中/待确认/已完成）
  - 表格：工单编号、标题、故障类型、业主、地址、状态、派单时间、操作
  - 分页
  - 操作：查看详情、接单（待处理）、开始维修（待处理）、完成维修（处理中）
- **交互**：接单、开始、完成可在列表或详情中操作

### 4.2 工单详情（维修工视角）
- **路径**：`/repairman/orders/detail/:id`
- **布局**：信息展示 + 操作区
- **内容**：
  - 基础信息：同业主详情
  - 业主信息：姓名、电话、地址
  - 处理过程：派单/接单/开始/完成时间、备注、图片、材料
  - 操作按钮：接单、开始维修、完成维修（按状态显示）
  - 完成维修弹窗：处理备注、处理图片上传、使用材料
- **交互**：完成维修时弹出表单，提交后刷新详情

---

## 五、管理员端页面

### 5.1 管理员首页 / 数据统计
- **路径**：`/admin/dashboard`
- **布局**：统计卡片 + 图表
- **内容**：
  - 顶部：统计卡片（工单总数、待处理数、已完成数、用户数等）
  - 报修类型分布：饼图或柱状图（typeName + count）
  - 工单完成率趋势：折线图（date、total、completed），支持 day/week/month
  - 维修工绩效排行：表格（姓名、完成数、平均评分）
- **交互**：日期范围、统计间隔可筛选

### 5.2 用户管理
- **路径**：`/admin/users`
- **布局**：表格 + 搜索 + 新增
- **内容**：
  - 搜索：角色、状态、关键词
  - 新增用户按钮
  - 表格：ID、用户名、姓名、手机号、角色、状态、创建时间、最后登录、业主/维修工信息、操作
  - 分页
  - 操作：修改状态（启用/禁用）
- **交互**：新增打开表单弹窗；修改状态用确认框

### 5.3 新增用户弹窗
- **内容**：
  - 基础：用户名、密码、姓名、手机号、角色、状态
  - 业主：楼栋号、单元号、房间号、面积、入住日期
  - 维修工：技能类型、工作年限
  - 确定、取消
- **交互**：角色切换时显示/隐藏对应扩展字段

### 5.4 公告管理
- **路径**：`/admin/announcements`
- **布局**：表格 + 筛选
- **内容**：
  - 筛选：状态下拉（草稿/已发布/已过期）
  - 表格：ID、标题、内容摘要、发布时间、过期时间、状态、置顶、浏览次数
  - 分页
- **说明**：当前接口仅支持查询，后续可扩展新增、编辑、发布

### 5.5 工单管理
- **路径**：`/admin/orders`
- **布局**：表格 + 多条件筛选
- **内容**：
  - 筛选：状态、优先级、业主、维修工、时间范围
  - 表格：工单编号、标题、故障类型、业主、维修工、状态、优先级、创建时间、操作
  - 分页
  - 操作：查看详情、派单（待派单）
- **交互**：派单打开弹窗，选择维修工并提交

### 5.6 派单弹窗
- **内容**：
  - 维修工选择：下拉（维修工列表，含姓名、技能、评分）
  - 确定、取消
- **交互**：提交后关闭弹窗并刷新列表

### 5.7 工单详情（管理员视角）
- **路径**：`/admin/orders/detail/:id`
- **布局**：同业主详情，增加派单入口
- **内容**：
  - 与业主详情相同
  - 待派单时显示「派单」按钮，打开派单弹窗

## 六、公共组件

### 6.1 消息通知
- **位置**：顶部导航栏右侧
- **内容**：
  - 铃铛图标 + 未读数量角标
  - 点击展开下拉列表：消息标题、内容摘要、时间、已读/未读
  - 底部：「全部已读」按钮
- **交互**：点击单条标记已读；支持跳转到消息列表页

### 6.2 消息列表页
- **路径**：`/notifications`
- **布局**：列表 + 分页
- **内容**：
  - 筛选：全部/未读/已读
  - 列表：类型图标、标题、内容、时间、已读状态
  - 操作：单条标记已读、全部已读
  - 分页

### 6.3 用户头像下拉
- **内容**：姓名、角色、退出登录
- **交互**：退出清除 token，跳转登录页

## 七、路由与权限

### 7.1 路由表（示例）
- `/login`：登录
- `/register`：注册
- `/owner/home`：业主首页
- `/owner/repair/create`：我要报修
- `/owner/repair/list`：我的报修
- `/owner/repair/detail/:id`：报修详情
- `/owner/repair/:orderId/evaluate`：提交评价
- `/repairman/orders`：维修工待办
- `/repairman/orders/detail/:id`：工单详情
- `/admin/dashboard`：数据统计
- `/admin/users`：用户管理
- `/admin/announcements`：公告管理
- `/admin/orders`：工单管理
- `/admin/orders/detail/:id`：工单详情
- `/notifications`：消息列表

### 7.2 权限控制
- 路由守卫：未登录跳转登录页
- 角色守卫：根据角色限制可访问路由
- 菜单：根据角色动态生成左侧菜单

## 九、状态与接口映射

### 9.1 登录后存储
- token
- userId、username、realName、role、avatar

### 9.2 请求封装
- 请求拦截：Header 添加 `Authorization: Bearer {token}`
- 响应拦截：401 跳转登录；业务错误用 `el-message` 提示

### 9.3 接口与页面对应
- 用户：`/api/users/*` → 登录、注册、用户管理
- 公告：`/api/announcements/*` → 首页公告、公告管理
- 故障类型：`/api/fault-types/tree` → 报修表单级联
- 工单：`/api/orders/*` → 报修、工单列表、详情、派单、接单、完成、取消
- 评价：`/api/orders/{id}/evaluation` → 评价表单、详情展示
- 统计：`/api/statistics/*` → 数据统计页
- 消息：`/api/notifications/*` → 消息铃铛、消息列表

---

以上为基于全部接口的 Vue + Element Plus 页面设计方案，可直接作为需求说明在另一窗口生成具体页面代码。
